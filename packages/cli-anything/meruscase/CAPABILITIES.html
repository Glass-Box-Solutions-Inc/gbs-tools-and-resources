Overview

What it does

MerusCase has no CLI and its web interface is inaccessible to AI agents. cli-anything-meruscase bridges that gap: it wraps MerusCase's REST API and browser-only workflows into a fully agent-native interface with structured JSON output, stateful REPL sessions, and a 50-level undo/redo stack.

📋

Case Management

List, search, and retrieve full case details. Create new cases via Browserless browser automation (bypasses reCAPTCHA).

💰

Billing Operations

Bill attorney time (hours → minutes conversion), add direct costs/fees/expenses, and pull billing summaries with totals.

📝

Activities & Notes

List all case activities. Add non-billable notes with subject, description, and activity type.

📄

Document Upload

Upload any local file to a case with optional description and folder targeting. List all existing case documents.

👥

Parties

List all parties on a case (Client, Employer, Insurance Company, Opposing Party, etc.) and add new parties.

🤖

Agent-Native

Every command outputs structured JSON. The REPL maintains session state across commands. Built for AI agent pipelines.

Getting Started

Installation

1

Install the package

pip install -e packages/cli-anything/meruscase/agent-harness
2

Install Playwright (for case creation only)

playwright install chromium

Only needed for case create. All other commands use the REST API.

3

Verify and authenticate

cli-anything-meruscase --help
cli-anything-meruscase auth login
✓
Token is loaded automatically from GCP Secret Manager, then the MERUSCASE_ACCESS_TOKEN env var, then ~/.meruscase_token. No manual setup required if GCP credentials are present.
Authentication

3-Tier Token Loading

The CLI resolves your MerusCase OAuth Bearer token automatically in priority order. No hard-coded credentials. Tokens are stored with chmod 0o600.

GCP Secret Manager

2
Environment variable

3
Local token file
~/.meruscase_token — written by cli-anything-meruscase auth login

Auth commands

cli-anything-meruscase auth login       # load or set token
cli-anything-meruscase auth status      # show token source + masked value
cli-anything-meruscase auth refresh     # force reload from GCP Secret Manager
Interface

Dual-Mode Interface

Interactive REPL (default)

Start without a subcommand. Maintains session state, undo/redo history, and a persistent prompt across commands. Ideal for exploratory use and agent sessions.

cli-anything-meruscase

  ╔══════════════════════════════╗
  ║   cli-anything-meruscase    ║
  ║   MerusCase CLI  v1.0.0     ║
  ╚══════════════════════════════╝

meruscase> case list --limit 5
meruscase> billing summary --case 4703360
meruscase> quit

Subcommand Mode

Pass a command directly. Returns immediately. Composable with shell pipelines, scripts, and AI agent tool calls. Combine with --json for structured output.

cli-anything-meruscase case list --limit 5

cli-anything-meruscase --json \
  case find "Smith" \
  | jq '.id'

cli-anything-meruscase --json \
  billing summary --case 4703360
ℹ
The --json flag is set at the root level and applies to every subcommand. Every command — including errors — emits machine-parseable JSON when the flag is set.

Examples

Common Workflows

Find a case and pull its billing summary

# Fuzzy find by name, pipe case ID into billing summary
CASE_ID=$(cli-anything-meruscase --json case find "Smith" | jq '.id')
cli-anything-meruscase --json billing summary --case $CASE_ID

Bill time and add a note in one session

cli-anything-meruscase
meruscase> billing bill-time --case 4703360 --hours 1.5 --desc "Reviewed QME report, drafted response letter"
meruscase> activity add-note --case 4703360 --subject "QME review complete" --desc "Sent letter to opposing counsel"
meruscase> session undo    # ← changed your mind
meruscase> quit

Upload a document

cli-anything-meruscase document upload \
  --case 4703360 \
  --file /path/to/QME_Report_Smith.pdf \
  --desc "QME Report — Dr. Johnson — 2026-04-14"

Create a new case (Browserless)

cli-anything-meruscase --json case create \
  --name "GARCIA, MARIA" \
  --type "Workers Compensation" \
  --date "03/15/2025"

{

Agent pipeline (JSON mode)

REST API

MerusCase Endpoints Used

All API calls go to https://api.meruscase.com with Authorization: Bearer {token}. Rate limiting is handled with exponential backoff (up to 3 retries).

Method Endpoint Purpose Notes
GET/caseFiles/indexList all casesPython-side filtering (API rejects query params)
GET/caseFiles/view/{id}Get case details
POST/caseFiles/edit/{id}Update case fields
GET/parties/index/{id}List case partiesCase ID as path segment (not query param)
POST/parties/addAdd a partyWrapped in {"Party": data}
GET/activities/indexList activitiesPython-side limit applied
POST/activities/addAdd activity/noteWrapped in {"Activity": data}
GET/activityTypes/indexActivity type reference1-hour in-process cache
GET/billingCodes/indexBilling code reference1-hour in-process cache
GET/caseLedgersOpen/indexLedger entriesDoubly-nested envelope normalized
POST/caseLedgers/addAdd fee/cost/expenseWrapped in {"CaseLedger": data}
POST/uploads/addUpload documentMultipart form-data
GET/uploads/indexList documents
⚠
MerusCase returns HTTP 200 even for errors — the errors field in the response body is always checked and raised as MerusCaseAPIError when present.
Browser Automation

Browserless Case Creation

New case creation has no REST API equivalent — it requires filling the MerusCase web form. cli-anything-meruscase handles this via Browserless, a cloud browser service that bypasses reCAPTCHA and works without a local display.

☁

Browserless Cloud

Connects to wss://production-sfo.browserless.io via CDP. No local Chrome needed. Token from GCP Secret Manager.

🛡

reCAPTCHA Bypass

Browserless auto-solves MerusCase's g-recaptcha-response hidden field before the login form renders.

🔗

Case ID Extraction

After form submission, the new case ID is extracted from the redirect URL via #/caseFiles/{ID} regex.

⚡

Async httpx

All REST calls are async. Exponential backoff on 429 rate-limit responses (base 2.0, up to 3 retries).

💾

50-Level Undo/Redo

Deep-copy snapshots before every mutating operation. New operations clear the redo stack. Stack trimmed at 50.

🔒

Credential Safety

Token files written with 0o600. Secrets never logged. PHI-safe: document IDs only, never content.
