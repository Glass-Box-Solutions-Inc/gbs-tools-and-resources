# Slack Notifier Module

**Status:** Implemented (Task 4.2)
**Module:** `intelligence/slack-notifier.js`
**Tests:** `tests/intelligence/slack-notifier.test.js`

## Overview

The Slack notifier sends daily intelligence briefings to the Slack #main channel via incoming webhook. It converts Gemini-generated markdown briefings into Slack Block Kit format for rich, structured messages.

## Features

### Core Functionality

- **Zero External Dependencies**: Uses native Node.js `https` module only
- **Slack Block Kit Formatting**: Converts markdown briefings to structured Slack messages
- **Automatic Retry Logic**: Exponential backoff on 5xx errors (3 attempts)
- **No Retry on 4xx**: Fast-fail on bad requests or invalid webhooks
- **Text Truncation**: Automatically truncates long text to fit Slack's 3000-char block limit
- **Graceful Degradation**: Notification failure does not fail intelligence pipeline
- **Timeout Handling**: 10-second timeout per request

### Block Kit Message Structure

Each briefing notification includes:

1. **Header Block**: Date and emoji
2. **Executive Summary**: Bullet points from Gemini synthesis
3. **Repository Activity**: Commit activity summary
4. **Pull Request Activity**: PR merge/open/close stats (if present)
5. **Infrastructure & Operations**: Deployment events
6. **Team Activity**: Claude Code sessions, Cursor activity
7. **Recommendations**: Gemini observations and suggestions
8. **Metrics Footer**: Commits, PRs, Deployments, Active Hours
9. **GitHub Link**: Link to full briefing in adjudica-documentation
10. **Footer Context**: Generated by Squeegee + model used

## API

### Main Function

```javascript
const { notify } = require('./intelligence/slack-notifier');

/**
 * Send intelligence briefing to Slack
 * @param {Object} briefing - Gemini briefing object (from gemini-synthesizer)
 * @param {Date|string} date - Date of the briefing (YYYY-MM-DD or Date object)
 * @param {Object} config - Configuration object
 * @returns {Promise<SlackNotificationResult>}
 */
const result = await notify(briefing, date, config);
```

### Configuration Schema

```javascript
{
  notifications: {
    slack: {
      enabled: true,                // Master toggle
      webhook_url: "https://...",   // Slack incoming webhook URL
      channel: "#main"               // Channel name (informational only)
    }
  }
}
```

### Result Object

```javascript
{
  success: true,                    // Whether notification sent successfully
  skipped: false,                   // True if notifications disabled
  channel: "#main",                 // Target channel
  timestamp: "2026-03-03T07:30:00Z", // When notification sent
  message_ts: "1234567890.123456",  // Slack message timestamp (if available)
  error: null                       // Error message if failed
}
```

## Usage

### Basic Usage

```javascript
const { notify } = require('./intelligence/slack-notifier');

const briefing = {
  executive_summary: [
    'Total of 15 commits across 5 repositories',
    '3 pull requests merged',
    '2 Cloud Run deployments successful'
  ],
  repository_activity: '...',
  deployment_events: '...',
  development_activity: '...',
  observations: '...',
  generated_at: '2026-03-03T07:15:00Z',
  model_used: 'gemini-2.0-flash-exp'
};

const config = {
  notifications: {
    slack: {
      enabled: true,
      webhook_url: process.env.SLACK_WEBHOOK_URL,
      channel: '#main'
    }
  }
};

const result = await notify(briefing, '2026-03-03', config);

if (result.success) {
  console.log('Briefing sent to Slack');
} else {
  console.error('Slack notification failed:', result.error);
}
```

### Disabled Notifications

```javascript
const config = {
  notifications: {
    slack: {
      enabled: false,  // Notifications disabled
      webhook_url: '...',
      channel: '#main'
    }
  }
};

const result = await notify(briefing, date, config);
// result.success === true, result.skipped === true
```

### Missing Webhook URL

```javascript
const config = {
  notifications: {
    slack: {
      enabled: true,
      webhook_url: null,  // Missing URL
      channel: '#main'
    }
  }
};

const result = await notify(briefing, date, config);
// result.success === false
// result.error === "Webhook URL not configured"
```

## Error Handling

### Retry Logic

**5xx Errors (Server Issues):**
- Retry 3 times with exponential backoff (2s, 4s, 8s)
- Example: Slack API temporarily unavailable

**4xx Errors (Client Issues):**
- No retry (fast-fail)
- Example: Invalid webhook URL, webhook revoked

**Network Errors:**
- Retry 3 times with exponential backoff
- Example: Connection timeout, DNS failure

### Graceful Degradation

Notification failures do NOT fail the intelligence pipeline:

```javascript
try {
  const result = await notify(briefing, date, config);
  // Always returns a result object, never throws
} catch (error) {
  // This should never happen - module catches all errors internally
}

// Intelligence pipeline continues even if Slack fails
```

## Slack Webhook Setup

### Create Incoming Webhook

1. Go to https://api.slack.com/apps
2. Select your workspace app (or create new app)
3. Navigate to **Incoming Webhooks**
4. Click **Add New Webhook to Workspace**
5. Select channel: `#main`
6. Copy webhook URL

## Troubleshooting

### Issue: Webhook returns 404

**Cause:** Webhook URL invalid or revoked

**Fix:**
1. Regenerate webhook in Slack
2. Update secret in GCP Secret Manager
3. Cloud Run will use new version automatically

### Issue: Webhook returns 400

**Cause:** Invalid Block Kit payload

**Fix:**
1. Check Block Kit structure in formatSlackMessage()
2. Validate against https://api.slack.com/block-kit/building
3. Use Block Kit Builder for testing

### Issue: Message truncated unexpectedly

**Cause:** Text exceeds Slack's 3000-char block limit

**Fix:**
- Already handled automatically via truncateText()
- Adjust maxLength parameter if needed

### Issue: Metrics not extracted correctly

**Cause:** Briefing format doesn't match regex patterns

**Fix:**
- Update extractMetrics() regex patterns
- Or manually pass metrics in briefing object
