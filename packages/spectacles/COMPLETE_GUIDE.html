What is Spectacles?

Spectacles is a cloud-deployed AI agent that provides visual and interactive capabilities that traditional AI systems lack. Think of it as "your AI's eyes" - it can see webpages, navigate sites, test interfaces, and coordinate development work across machines.

🚀 NEW: AI Reasoning Engine
Spectacles now learns from experience! After executing a task once, it remembers the successful pattern and can repeat it 5-10x faster with 80-95% cost reduction. See AI Reasoning Engine section below.

Spectacles serves two complementary roles, both centered on providing visual and interactive capabilities:

🌐 Mode 1: Direct Browser Automation

"I need to see or interact with a webpage"

  • Screenshots and visual verification
  • Testing user flows (login, checkout)
  • Verifying responsive layouts
  • Managing web-based credentials
  • Navigating complex OAuth flows

💻 Mode 2: Claude Code Coordination

"I need to write code AND verify it visually"

  • Code generation with visual verification
  • Remote control of development machines
  • Long-running development tasks
  • Multiple file modifications
  • Coordinated work across projects

AI Reasoning Engine

Spectacles 2.0 introduces an intelligent learning system that transforms browser automation from a one-off task into a continuously improving capability.

What It Does

🧠 Pattern Learning

After successfully completing a task once, Spectacles automatically:

  • Classifies the task type (LOGIN_FLOW, FORM_STRUCTURE, NAVIGATION, EXTRACTION)
  • Extracts element selectors and action sequences
  • Stores the pattern with semantic embeddings in Qdrant
  • Calculates initial confidence based on task success

⚡ Intelligent Execution

When a similar task arrives, Spectacles:

  • Searches for matching patterns via semantic similarity
  • Evaluates pattern confidence (threshold: 0.7)
  • Uses pattern directly if confident (0 VLM calls)
  • Falls back to VLM discovery if no match or low confidence

Confidence Tracking

Pattern confidence is calculated using a sophisticated formula that considers:

  • Success Rate: Proportion of successful executions
  • Sample Size: Patterns need 10+ uses for maximum confidence
  • Recency: Patterns older than 30 days receive a staleness penalty
confidence = base - sample_penalty - staleness_penalty

base = success_count / (success_count + failure_count)
sample_penalty = 0.1 × (1 - total/10) if total < 10
staleness_penalty = min(0.3, 0.01 × (days - 30)) if days > 30

Confidence Examples:

Scenario Stats Confidence Verdict
New pattern 3 successes, 0 failures, 1 day old 0.93 ✅ Good but unproven
Mature pattern 20 successes, 0 failures, 5 days old 1.0 ✅ Perfect
Occasional failures 40 successes, 10 failures, 10 days old 0.8 ✅ Reliable
Stale pattern 10 successes, 0 failures, 90 days old 0.7 ⚠️ Needs revalidation

Security & Privacy

Credentials Management

How it works:

  • Credentials stored in GCP Secret Manager (encrypted at rest)
  • Referenced by credentials_key (e.g., "google_oauth")
  • NEVER in LLM context - injected at execution time only
  • Encrypted browser session storage

Example:

{
  "goal": "Log in and check dashboard",
  "credentials_key": "app_admin"
}
Spectacles retrieves credentials from Secret Manager, uses them for login, never exposes them to AI.

PII Protection

Screenshots are processed before Slack:

  • Credit card numbers → blurred
  • SSN patterns → blurred
  • Phone numbers → blurred
  • Email addresses → optionally blurred
Example: If Spectacles screenshots a payment form, card numbers are automatically detected and obscured before sending to Slack.

Audit Logging

All actions are logged:

  • Task submissions (who, what, when)
  • Browser actions (navigation, clicks, form fills)
  • HITL interactions (approvals, rejections)
  • Security events (failed authentication, permission denied)
  • Pattern learning events (patterns learned, confidence updates)

Logs stored in: GCP Cloud Logging (90-day retention)

Support & Troubleshooting

Common Issues

Issue 1: "Permission denied" (Mode 2)

Problem: You ran /spectacles ask and got "Permission denied"

Solution:

  1. Request admin access (see "How to Get Access" section)
  2. Verify your Slack User ID is in CLAUDE_CODE_ADMIN_USERS
  3. Wait for admin to redeploy Spectacles

Issue 2: Browser timeout

Problem: Task stuck in "Navigating" state for >5 minutes

Solution:

  1. Check if URL is accessible (try loading in your browser)
  2. Verify credentials_key is correct (if using authentication)
  3. Cancel task and retry with simpler goal
  4. Contact admin if issue persists

Issue 3: Pattern not being used

Problem: Task is still using VLM discovery mode even though you've run it before

Solution:

  1. Check pattern confidence via API: GET /api/patterns/
  2. Confidence may be below 0.7 threshold (needs more successful runs)
  3. Pattern may be too old (staleness penalty applied)
  4. Task may be different enough that semantic matching didn't find it

Issue 4: Screenshot blank

Problem: Mode 1 returns blank screenshot

Solution:

  1. Verify URL loads in browser (might be behind login)
  2. Add credentials_key if authentication required
  3. Try with "wait_until": "domcontentloaded" (faster but less complete)
  4. Check if page has anti-bot protection

State Machine (Mode 1)

The browser automation agent operates through these states:

  1. PLANNING - Analyzes goal, queries pattern store, creates action plan
  2. NAVIGATING - Navigates to target URL
  3. OBSERVING - Perceives page state (DOM or Vision)
  4. ACTING - Executes browser action
  5. EVALUATING - Checks if goal achieved
  6. AWAITING_HUMAN - Paused for Slack response
  7. ERROR_RECOVERY - Handling failures
  8. LEARNING - Extracting patterns (NEW!)
  9. COMPLETED - Task finished, pattern stored
